# 快速上手

## 使用命令行

创建项目:

```sh
pnpm create preezid-app
```

启动开发服务器:

```sh
pnpm run dev
```

构建生产版本:

```sh
pnpm run build
```

## 使用 CDN

由于浏览器默认不支持 `lts` 拓展语法, 因此在使用 `CDN` 时, 我们需要引入 `Preezid` 的前置: `Preezid Compiler` 和 `PiteRender`.

下面简单介绍一下 `PiteRender`.

`PiteRender` 是 `Preezid` 函数式的前一次尝试, 它同样使用函数式编程的方式来构建界面, 但它并没有自己的 DSL, 只能使用纯 `js/ts` 实现, 因此代码冗长.

下面是一个使用 `PiteRender` 的简单示例:

```ts
render("div")
  .class("container")
  .style({
    backgroundColor: "#00aaff",
  })
  .slot([
    () => {
      arr.forEach(item => {
        return render("span").slot(item.name).build()
      })
    },
    () => {
      for (const item of arr) {
        return render("span").slot(item.name).build()
      }
    },
    render("button").event("onclick", handleClick).slot("Click me !").build(),
  ])
  .build()
```

`PiteRender` 封装了一个 `render` 类, 该类使用一系列链式调用来构建页面, 它完全是 `js/ts` 的一个库, 虽然可以直接在浏览器中使用, 但无法编写脱离 `js/ts` 的代码, 实际使用体验极差.

不过尽管如此, `PiteRender` 依旧成为了 `Preezid` 的基础, 并集成到了 `Preezid` 中, 作为其底层渲染引擎.

`Preezid` 的编译运行过程就是将 `ljs/lts` 的 DSL 编译为 `PiteRender JS/TS`, 之后直接运行 `PiteRender` 代码来完成渲染.

而 `Preezid Compiler` 则是将 `ljs/lts` DSL 编译为 `PiteRender JS/TS` 的工具.

因此, 如果你想在浏览器中使用 `Preezid`, 你需要引入 `Preezid`, 它包含 `Preezid Compiler` 和 `PiteRender`:

此外, `ljs/lts` 实际上也包含若干自动导入的内容, 在 `Preezid` 项目中可直接使用, 但 CDN 导入无法实现该功能, 因此你需要手动导入这些内容.

```html
<script src="https://cdn.preezid.com/preezid@latest/preezid.js" crossorigin></script>

<script type="text/ljs">
    const root = preezid.init(document.getElementById("root"));
    preezid.load(root, "preezid", () => {
        div.class("container") {
            button.@click(() => alert("Hello Preezid with CDN!")) { "Click me !" }
        }
    })
</script>
```
